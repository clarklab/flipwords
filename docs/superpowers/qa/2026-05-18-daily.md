### 3. Session completion writes storage

**PASS**

`play.tsx` line 30–34: `handleComplete` calls `recordCompletion(loadStorage(), startDate, result)`,

`recordCompletion` in `streak.ts` runs `tickStreak` then merges the session and
updates totals. `saveStorage` writes to `localStorage`.

`Scorecard.tsx` lines 145–181: when `streak` prop is non-null, renders the streak
chip; `{streak.deltaThisSession && (<span>+1 today</span>)}` renders the pill.

### 10. `/archive/$date` does NOT persist

**PASS**

`src/routes/archive.$date.tsx` is present and renders `<FlipWords mode="archive">`
**without an `onComplete` prop**. FlipWords' `if (onComplete && ...)` guard means
the `recordCompletion` write path never runs. The scorecard primary CTA is
"Back to archive" → `navigate({ to: '/archive' })`. The route also guards against
`date < LAUNCH_DATE` or `date > today`, rendering a "No puzzle for that date."
fallback in those cases.

### 11. `settleStreak` fires on TitleScreen mount

**PASS**

`TitleScreen.tsx` lines 38–42:
```ts
useEffect(() => {
  const settled = settleStreak(loadStorage(), today)
  saveStorage(settled)
  setStorage(settled)
}, [today])
```
`settleStreak` resets `current` to 0 if the last completed date is more than one
day behind today (i.e., a streak was broken). The effect runs once on mount (and
again if the Eastern date rolls over while the page is open).

### 12. Cross-midnight rule

**PASS**

`play.tsx` line 21: `const [startDate] = useState(() => easternDateString())`.
`useState` initialiser runs exactly once — on first render. `startDate` is the
Eastern date at the moment the user navigated to `/play`. Lines 31 and 33 use
`startDate` as the persistence key, so a session that spans midnight still
writes to the original day's key.
